# Buttons ![Ready Badge](https://img.shields.io/badge/component-ready-green.svg#badge)

Buttons are used to clearly communicate clear action to the user.

## General Usage Guidelines

- Buttons should be used for important action items like submitting forms, saving pages, or completing purchases
- Buttons can also indicate the beginning of a new task or the next steps in a multi-step process
- Buttons can also be used when creating new entities like new sites, new blog posts, or new products

### Label

Button labels should be verbs that connote action like Save, Delete, or Complete Purchase. 

### Standard (Primary) Button 

- The strongest calls to action on the page.
- Try to use only one primary call to action per page.
- Not all pages require a primary call to action.

<Playground>
  <ButtonToolbar>
    <Button bsStyle="primary">Standard</Button>
    <Button bsStyle="primary" active>Active</Button>
    <Button bsStyle="primary" disabled>Disabled</Button>
  </ButtonToolbar>
</Playground>

### Outline (Secondary) Button 

- Use when the pages primary action is not obvious or when there are complimentary actions the user can perform.
- There can be multiple outline/secondary buttons on a single page.

<Playground>
  <ButtonToolbar>
    <Button>Outline</Button>
    <Button active>Active</Button>
    <Button disabled>Disabled</Button>
  </ButtonToolbar>
</Playground>

### Link Button

- Use when trying to give an action the least possible emphasis.
- Use when there are many actions available and you are trying not to saturate the page with buttons. 
- Link buttons are not hyperlinks in a pararaph of text, they are to be used like buttons.
- There can be multiple link buttons on a single page.

<Playground>
  <ButtonToolbar>
    <Button bsStyle="link">Standard</Button>
    <Button bsStyle="link" active>Active</Button>
    <Button bsStyle="link" disabled>Disabled</Button>
  </ButtonToolbar>
</Playground>

### Destructive Button

- Use whenever a destructive action is present on a screen. 
- These buttons are typically used in modals whereby the user must confirm the destructive action.

<Playground>
  <ButtonToolbar>
    <Button bsStyle="danger">Danger</Button>
    <Button bsStyle="danger" active>Active</Button>
    <Button bsStyle="danger" disabled>Disabled</Button>
  </ButtonToolbar>
</Playground>

> Button spacing

> Because React doesn't output newlines between elements, buttons on the same line are displayed flush against each other. To preserve the spacing between multiple inline buttons, wrap your button group in `<ButtonToolbar />`.

## States

### Block Level

Create block level buttons—those that span the full width of a parent— by adding the block prop.

<Playground>
  <Button bsStyle="primary" block>Block level button</Button>
  <Button block>Block level button</Button>
</Playground>

## Button tags

The DOM element tag is choosen automatically for you based on the props you supply. Passing a href will result in the button using a `<a />` element otherwise a `<button />` element will be used.

<Playground>
  <ButtonToolbar>
    <Button href="#">Link</Button>
    <Button>Button</Button>
  </ButtonToolbar>
</Playground>

### Checkbox / Radio

For checkboxes or radio buttons styled as Bootstrap buttons, use the `<ToggleButtonGroup>` and `<ToggleButton>` components. The group behaves as a form component, where the value is an array of the selected eventKeys for checkbox groups or the selected eventKey for radio groups.

> Bootstrap JS issue

> There is a known issue when including Bootstrap JS while using Toggle Button Groups. Therefore it is advised not use Bootstrap JS in conjunction with `<ToggleButtonGroup>` and `<ToggleButton>`.
