# Variables

> Building blocks for custom CSS with Bluebird

## Introduction

Keeping a wide array of applications in visual harmony requires guidelines and patterns, but sometimes Component style building blocks are not enough to achieve the spirit of a design. For Bluebird, that's where _Variables_ (also known as _abstracts_) come into play.

Variables are micro-sized building blocks that enable you to write custom css to achieve the designs you are provided. As you write custom css you begin to own the maintenance of your design and application, but by using variables you can offload some of that maintenance to Bluebird.

As an example, you may be designing a series of paragraphs where each has a set amount of space below it. You could write something like:

```scss
p {
  margin-bottom: 16px;
}
```

The problem with this approach is that when the UX team decides to move to a different vertical spacing standard, you must _manually_ update your styles to match.

Bluebird provides spacing, typography and many other variables to use throughout your custom stylesheets that will allow you to offset the maintenance burden.

The previous example can be improved with:

```scss
p {
  margin-bottom: $sm; // 16px
}
```

By using the scss variable `$sm`, the component adopts the UX standard from Bluebird, and if the design system later changes, so will your component.
