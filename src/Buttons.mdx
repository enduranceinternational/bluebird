# Buttons

Buttons are UI elements that clearly communicate to the user that they can complete an action.
- Buttons should indicate that they trigger an action
- Buttons should be easy to find
- A buttons action and state should be clear

## Usage

Use any of the available button style types to quickly create a styled button. Just modify the bsStyle prop.

**Primary** - Primary buttons draw the users attention to a specific action

<Playground>
  <ButtonToolbar>
    <Button bsStyle="primary">Primary</Button>
  </ButtonToolbar>
</Playground>

**Secondary** - Secondary buttons emphasize actions that are less important

<Playground>
  <ButtonToolbar>
    <Button>Secondary</Button>
  </ButtonToolbar>
</Playground>

**Destruction** - Destructive buttons express caution to users and are tied to actions like delete or deactivate

<Playground>
  <ButtonToolbar>
    <Button bsStyle="danger">Destructive</Button>
  </ButtonToolbar>
</Playground>

**Text** - Text buttons emphasize actions that are the least important

<Playground>
  <ButtonToolbar>
    <Button bsStyle="link">Link</Button>
  </ButtonToolbar>
</Playground>

**Small** - Small buttons ___

<Playground>
  <ButtonToolbar>
    <Button bsStyle="primary" bsSize="small">Small button</Button>
  </ButtonToolbar>
</Playground>



> Button spacing

> Because React doesn't output newlines between elements, buttons on the same line are displayed flush against each other. To preserve the spacing between multiple inline buttons, wrap your button group in `<ButtonToolbar />`.

## States


### Block level

Create block level buttons—those that span the full width of a parent— by adding the block prop.

<Playground>
  <Button bsStyle="primary" bsSize="large" block>
    Block level button
  </Button>
  <Button bsSize="large" block>
    Block level button
  </Button>
</Playground>

## Active state

To set a buttons active state simply set the components active prop.

<Playground>
  <ButtonToolbar>
    <Button bsStyle="primary" bsSize="large" active>
      Primary button
    </Button>
    <Button bsSize="large" active>
      Button
    </Button>
  </ButtonToolbar>
</Playground>

## Button tags

The DOM element tag is choosen automatically for you based on the props you supply. Passing a href will result in the button using a `<a />` element otherwise a `<button />` element will be used.

<Playground>
  <ButtonToolbar>
    <Button href="#">Link</Button>
    <Button>Button</Button>
  </ButtonToolbar>
</Playground>

## Disabled state

Make buttons look unclickable by fading them back 50%. To do this add the disabled attribute to buttons.

<Playground>
  <ButtonToolbar>
    <Button bsStyle="primary" bsSize="large" disabled>
      Primary button
    </Button>
    <Button bsSize="large" disabled>
      Button
    </Button>
  </ButtonToolbar>
</Playground>

## Button groups

Group a series of buttons together on a single line with the button group.

### Basic Example

Wrap a series of `<Button />`s in a `<ButtonGroup />`.

<Playground>

  </ButtonGroup>
</Playground>

### Nesting

You can place other button types within the `<ButtonGroup />` like `<DropdownButton />`s.


### Vertical variation

Make a set of buttons appear vertically stacked rather than horizontally. *Split button dropdowns are not supported here.*

Just add `vertical` to the `<ButtonGroup />`.

<Playground>
  <ButtonGroup vertical>
    <Button>Button</Button>
    <Button>Button</Button>
    <DropdownButton title="Dropdown" id="bg-vertical-dropdown-1">
      <MenuItem eventKey="1">Dropdown link</MenuItem>
      <MenuItem eventKey="2">Dropdown link</MenuItem>
    </DropdownButton>
    <Button>Button</Button>
    <Button>Button</Button>
    <DropdownButton title="Dropdown" id="bg-vertical-dropdown-2">
      <MenuItem eventKey="1">Dropdown link</MenuItem>
      <MenuItem eventKey="2">Dropdown link</MenuItem>
    </DropdownButton>
    <DropdownButton title="Dropdown" id="bg-vertical-dropdown-3">
      <MenuItem eventKey="1">Dropdown link</MenuItem>
      <MenuItem eventKey="2">Dropdown link</MenuItem>
    </DropdownButton>
  </ButtonGroup>
</Playground>

Moreover, you can have buttons be block level elements so they take the full width of their container, just add `block` to the `<ButtonGroup />`, in addition to the vertical you just added.

<Playground>
  <ButtonGroup vertical block>
    <Button>Full width button</Button>
    <Button>Full width button</Button>
  </ButtonGroup>
</Playground>

### Justified button groups

Make a group of buttons stretch at equal sizes to span the entire width of its parent. Also works with button dropdowns within the button group.

> Style issues

> There are some issues and workarounds required when using this property, please see bootstrap’s button group docs for more specifics.

Just add `justified` to the `<ButtonGroup />`.

<Playground>
  <ButtonGroup justified>
    <Button href="#">Left</Button>
    <Button href="#">Middle</Button>
    <DropdownButton title="Dropdown" id="bg-justified-dropdown">
      <MenuItem eventKey="1">Dropdown link</MenuItem>
      <MenuItem eventKey="2">Dropdown link</MenuItem>
    </DropdownButton>
  </ButtonGroup>
</Playground>

### Checkbox / Radio

For checkboxes or radio buttons styled as Bootstrap buttons, use the `<ToggleButtonGroup>` and `<ToggleButton>` components. The group behaves as a form component, where the value is an array of the selected eventKeys for checkbox groups or the selected eventKey for radio groups.

> Bootstrap JS issue

> There is a known issue when including Bootstrap JS while using Toggle Button Groups. Therefore it is advised not use Bootstrap JS in conjunction with `<ToggleButtonGroup>` and `<ToggleButton>`.

#### Uncontrolled

<Playground>
  <ButtonToolbar>
    <ToggleButtonGroup type="checkbox" defaultValue={[1, 3]}>
      <ToggleButton value={1}>Checkbox 1 (pre-checked)</ToggleButton>
      <ToggleButton value={2}>Checkbox 2</ToggleButton>
      <ToggleButton value={3}>Checkbox 3 (pre-checked)</ToggleButton>
    </ToggleButtonGroup>
  </ButtonToolbar>

  <ButtonToolbar>
    <ToggleButtonGroup type="radio" name="options" defaultValue={1}>
      <ToggleButton value={1}>Radio 1 (pre-checked)</ToggleButton>
      <ToggleButton value={2}>Radio 2</ToggleButton>
      <ToggleButton value={3}>Radio 3</ToggleButton>
    </ToggleButtonGroup>
  </ButtonToolbar>
</Playground>

#### Controlled

<Playground>
  {() => {
    class ToggleButtonGroupControlled extends React.Component {
      constructor(props, context) {
        super(props, context);

        this.handleChange = this.handleChange.bind(this);

        this.state = {
          value: [1, 3]
        };
      }

      handleChange(e) {
        this.setState({ value: e });
      }

      render() {
        return (
          <ToggleButtonGroup
            type="checkbox"
            value={this.state.value}
            onChange={this.handleChange}
          >
            <ToggleButton value={1}>Checkbox 1 (pre-checked)</ToggleButton>
            <ToggleButton value={2}>Checkbox 2</ToggleButton>
            <ToggleButton value={3}>Checkbox 3 (pre-checked)</ToggleButton>
            <ToggleButton value={4} disabled>
              Checkbox 4 (disabled)
            </ToggleButton>
          </ToggleButtonGroup>
        );
      }
    }

    return <ToggleButtonGroupControlled />;
  }}
</Playground>
